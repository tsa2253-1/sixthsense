Mission & Ethics

About SixthSense

Built on empathy-driven engineering. Grounded in universal design principles. Free for every person on every device, forever.

Ethical Design Statement

SixthSense was built on one belief: technology should work for every person, regardless of their sensory ability, economic status, or technical background. Accessibility is not a feature. It is the foundation.

We follow Universal Design — building for the full range of human diversity from the start, not as an afterthought. Every interaction, every colour, every font size, every button label was chosen with a specific user need in mind.

This application collects zero personal data, shows zero advertisements, and requires zero sign-up. All preferences live in your browser and never leave your device.

The Sixth Sense Concept

The five human senses — sight, hearing, touch, taste, smell — evolved over millions of years to help us navigate the world. For individuals with hearing or vision impairments, one of those channels is reduced or absent — but the need to perceive the world isn't.

SixthSense repurposes the computer as a perceptual prosthetic — a sixth sense that bridges the gap between what the body provides and what the environment demands. Speech becomes text. Text becomes speech. Visual scenes become audio descriptions. Silence becomes vibration.

The technology is not the point. Independence is.

Accessibility Impact by Feature

Each SixthSense feature, its primary audience, the barrier it removes, and the technology used
Feature Primary Audience Barrier Removed Technology
SpeakAssist Deaf, hard-of-hearing Cannot access spoken content in real time Web Speech API — SpeechRecognition
ReadAssist Low vision, dyslexia, reading fatigue Small fonts, poor contrast, text inaccessibility CSS transforms + SpeechSynthesis API
VisionAssist Blind, low vision Cannot read printed text, signs, or documents MediaDevices API + Tesseract.js (client-side OCR)
AdvancedAlert Deaf, blind, and all users Audio-only alarms exclude hearing-impaired Vibration API + SpeechSynthesis + CSS animation
Control Panel All users One-size-fits-all interfaces fail diverse needs localStorage + CSS custom properties
